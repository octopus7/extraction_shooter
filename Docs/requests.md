## 2026-05-10 17:40:59 (소요시간: 11분 10초)

- Enhanced Input 기반 탑다운 캐릭터 이동을 구현할 것.
- 에디터 모듈로 `IMC_`, `IA_` 입력 asset을 생성할 것.
- WASD 이동, 마우스 커서 방향 플레이어 회전, LMB 홀드 연사, RMB 홀드 조준 상태를 구현할 것.
- 조준 중 카메라 FOV를 좁히고 사선 방향으로 카메라 중심을 보정하되, 카메라는 회전하지 않게 할 것.
- 플레이어 BP는 임시 Static Mesh Cylinder로 만들고, 추후 Skeletal Mesh 연결을 고려할 것.
- 투사체 BP와 무기 BP를 만들고 플레이어에 무기를 attach할 것.
- 잔탄, 피해량 같은 게임 로직 없이 이동, 조준, 발사, 연발만 구현할 것.
- 적 BP를 만들고 AI는 AIController C++ 코드로 동작하게 하며, Behavior Tree는 프로젝트에서 사용하지 않을 것.
- Behavior Tree 미사용 방침을 `README.md`에 기록할 것.

## 2026-05-10 18:48:32 (소요시간: 12분 44초)

- 상호작용 가능한 Actor 클래스를 만들고, 실제 동작 종류와 UI 표시 명칭은 BP에서 지정할 수 있게 할 것.
- 테스트용 상호작용 항목으로 대화, 줍기, 열기 BP를 만들고 레벨에 배치할 것.
- UMG 스크린 스페이스 마커를 만들고, 표시 가능 거리 안에서는 페이드 인, 밖에서는 페이드 아웃되게 할 것.
- 마커의 원형 링은 표시 가능 거리 진입 시 스케일 3배에서 1배로 줄고, 이탈 시 1배에서 3배로 커지며 투명해지게 할 것.
- 마커 오른쪽에는 흰 바탕 사각형과 검은색 표시 명칭 텍스트를 표시할 것.
- 마커는 이후 위젯 에디터에서 수정 가능한 `WBP_` 에셋으로 만들 것.

## 2026-05-10 19:09:19 (소요시간: 4분 51초)

- 상호작용 액터 마커의 우측 텍스트는 상호작용 가능 거리 이내의 인접 액터 중 가장 가까운 액터 하나만 페이드되며 표시되게 할 것.
- 상호작용 처리는 액터가 직접 처리하지 않고 수신하는 서브시스템에게 위임할 것.
- 아직 실제 게임 동작은 붙이지 않고, 대화/줍기/열기 3가지 동작은 스크린 스페이스 디버그 로그로만 표시할 것.
- 이후 실제 동작 처리를 붙일 수 있도록 구조를 준비할 것.
