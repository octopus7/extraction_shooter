### 질문

벙커 레이드 전환간 동영상 재생하고 싶은데 어떻게 할까? 과정 하나씩 알려줘

### 답변

레벨 전환 액터가 바로 `OpenLevel`을 호출하지 않고, 전환 위젯에서 동영상을 재생한 뒤 재생 종료 이벤트에서 목표 레벨로 이동하는 구조가 적합하다.

1. `TunaSweeper/Content/Movies`에 전환용 MP4를 넣는다.
2. `FileMediaSource`를 전환 방향별로 만든다. 예: `MS_BunkerToRaid`, `MS_RaidToBunker`.
3. 공용 `MediaPlayer`와 `MediaTexture`를 만든다. 예: `MP_LevelTransition`, `MT_LevelTransition`.
4. 전체 화면 영상 표시용 `WBP_LevelTransitionVideo`를 만든다.
5. `UTunaSweeperLevelTransitionWidget`에서 MediaPlayer 재생, 종료 이벤트 수신, 종료 후 `OpenLevel(TargetLevelName)` 호출을 처리한다.
6. `ATunaSweeperLevelTravelInteractableActor`는 목표 레벨과 MediaSource를 전환 위젯에 넘기고, 직접 레벨을 열지 않도록 바꾼다.
7. 영상 로드나 재생 실패 시에는 바로 `OpenLevel`로 fallback한다.
8. 영상 재생 중에는 입력을 차단해 중복 상호작용을 막는다.

### 답변

엄폐물 인지는 적이 장애물을 단순히 피하는 기능이 아니라, 전투 중 선택 가능한 전술 위치를 평가하는 기능으로 잡는 게 좋다. UE 5.7 기준으로는 `AI Perception`이나 현재 타겟 인지 로직으로 플레이어 위치를 알고, `EQS` 또는 직접 생성한 엄폐 후보 지점 목록에서 플레이어와 후보 지점 사이의 라인트레이스가 막히는지, 적이 그 지점까지 갈 수 있는지, 사격 거리와 이동 비용이 적절한지 평가하는 구조가 적합하다. 최소 구현은 장애물 주변에 `CoverPoint` 액터나 컴포넌트를 배치하고, 전투 중 가까운 엄폐점을 점수화해 이동한 뒤 짧게 피킹/사격/재장전/재배치하는 상태를 두는 방식이다. 이후에는 피격 직후 엄폐 선호도 증가, 점유된 엄폐점 제외, 너무 자주 옮기지 않는 쿨다운, 엄폐점마다 좌/우 피킹 방향과 노출도를 추가하면 정직하게 직선 추적하는 느낌이 크게 줄어든다.

### 답변

원인은 드롭 이벤트를 실제로 처리하는 위치가 달랐기 때문이다. 슬롯 위젯은 `NativeOnDragOver()`를 받아서 받을 수 있는 슬롯 하이라이트를 켰지만, 마우스 버튼을 뗄 때의 `NativeOnDrop()`은 항상 그 슬롯 위젯으로 들어오지 않고 `TileView`나 부모 인벤토리/루팅 패널 쪽으로 빠지는 경우가 있었다. 부모 쪽 `NativeOnDrop()`에는 실제 이동 호출이 없어서 hover 중에는 하이라이트가 보이지만 drop 시에는 아무 이동도 일어나지 않았다. 수정은 hover 중 마지막 유효 슬롯을 `DragDropOperation`에 기록해 두고, drop이 부모로 들어와도 그 슬롯으로 `MoveItemBetweenSlots()`를 호출하게 한 것이다.

### 질문

### 답변

처음부터 N프레임까지만 남기려면 `-frames:v N`을 쓰면 된다. 예를 들어 처음부터 300프레임만 남기려면 `ffmpeg -i input.mp4 -frames:v 300 -c:v libx264 -crf 18 -preset medium -c:a aac -shortest output.mp4`처럼 실행한다. 프레임 번호를 0부터 세는 인덱스 기준으로 특정 프레임까지 포함하고 싶다면 `select='lte(n\,FRAME_INDEX)'` 필터를 써서 정확히 지정할 수 있다.

세이브 파일 이름이 `TunaSweeperInventoryState_Slot1`처럼 인벤토리 전용으로 되어 있는데, 앞으로 퀘스트나 레벨 언락 등 여러 정보를 저장해야 한다. 슬롯당 한 파일에 모두 넣는 게 맞는지, 슬롯 폴더 안에 데이터 종류별 여러 파일을 넣는 게 맞는지, 여러 파일을 압축하는 게 맞는지 의견은 무엇인가?

### 답변

지금 단계의 로컬 싱글플레이 게임이라면 슬롯당 한 `SaveGame` 파일에 모두 넣는 방식이 가장 적합하다. 파일 이름은 `TunaSweeperSave_Slot01` 또는 `TunaSweeperProfile_Slot01`처럼 인벤토리 전용 의미를 빼는 편이 좋다. 한 파일 안에 인벤토리, 퀘스트, 언락, 플레이 통계, 월드 상태를 구조체로 나누어 저장하면 저장/로드 시점의 일관성을 유지하기 쉽고, 백업/삭제/클라우드 세이브 처리도 단순하다. 데이터 종류별 여러 파일이나 압축은 대형 월드 청크, 리플레이, 스크린샷 같은 큰 데이터가 생긴 뒤에 고려해도 늦지 않다.
